# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| latest `main` | ✅ |
| older releases | ❌ |

## Reporting a Vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

This library is used in financial applications. Please report responsibly.

Email: **dev@decentralchain.io**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix

You will receive a response within 5 business days. We aim to release a patch
within 30 days of confirmation.

## Scope

This policy covers the `io.decentralchain:wavesj` library itself. Vulnerabilities
in upstream dependencies (`com.wavesplatform:lang`) should also be reported here
so we can coordinate a patched release. Vulnerabilities in
`io.decentralchain:transactions-java` (sibling package) should be reported via
the same email.

For the upstream `wavesplatform/WavesJ`, report to
[wavesplatform/WavesJ](https://github.com/wavesplatform/WavesJ/security).
